Stage 0: The Basics

The World Wide Web

The web is made up of Billions of HTML (HyperText Markup Language) documents connected through Hyperlinks. The content on the Internet is stored on servers that can be accessed through a web browser.

A web browser communicate with servers through HTTP (HyperText Transfer Protocol) by entering a URL (Uniform Resource Locator). The server will in turn respond with a document which the browser interprets and displays.

HyperText Markup Language

HTML is the language used to visualize the content of the web by a common standard. A HTML document is made up of:

  • Text content - (What you see)
  • Markup - (What it looks like)
  • References to other documents - (e.g. images and videos)
  • Links to other pages - (Hyperlinks)

Tags and Elements

HTML code consists of elements containing tags. There's an opening and closing tag. The structure looks like this:

<opening-tag>content</closing-tag>

This constitutes an element. Void tags are the exception as they don't require a closing tag. An example of this is the <br> tag.

Attributes

Tags can also contain Attributes which can, for example, create a link to a different web-page. The Anchor tag is an example of this: <a href="www.udacity.com"> where href is the Attribute and "www.udacity.com" is the value.

Inline and Block Elements

HTML elements are divided into two different categories: Inline and Block. Inline elements are, as the name suggests; in line, and Block elements have an "invisible box" around them.

How computers interprets code

When code, such as HTML, is rendered by a web browser it's done so literally. In other words; the computer is not applying any creativity when it "reads" code to fill in blanks or correct for errors (as a human might do). It also means that it will do exactly what the code tells it to do.

Stage 1: HTML and CSS

Structure

HTML and CSS (Cascading Style Sheets) are two Languages that work together to bring structure and style, respectively, to a website (JavaScript brings the action). HTML is structured with tags through something called DOM (Document Object Model).

The structure of HTML is much like a family tree, where elements branches out into more elements and form relations. Parent elements can have child elements and so on. These relations have significance as to how rules are inherited throughout the structure.

The underlying structure of HTML is made up of boxes. When building a HTML document it's best practice to start by outlining the "box structure" first. The most fundamental building block in this structure is the <div> element.

Style

In programming it's important to avoid repetition, not only to save time but also to minimize mistakes. By using CSS you can simplify and streamline the styling process by applying the same features to many different elements at the same time.

Inheritance is one of the key features of CSS. In general, text properties are inherited from parent elements while box elements are not.

In CSS you work with Selectors and Declarations. Declarations are broken down into Properties and Values : selector { property: value; }.

There are many different types of Selectors you can work with in CSS. A good explanation of how they work and what they do can be found here. However, one important function of CSS is that you can precisely specify which HTML element(s), rules are applied to depending on which type of Selector you use.

Stage 2: Programming, Computer Science and Python

Programming as a Language

Everything that a computer executes is done through a program. A program is an instruction with a set of rules that the computer can understand and follow, written in a programming language.

The reason we don't use a natural language in programming, is that there's generally too much ambiguity regarding the meaning of a word or a sentence. Also, a natural language tend to be very verbose.

The Backus-Naur Form (BNF) is a notation technique to describe the language so that the rules of interpretation are unambiguous. By using this method for programming languages, the programmer and computer can communicate in an equitable way as well as making sure that the computer understands and executes the code correctly. BNF is like grammar in a natural language. It's a set of rules to describe how we apply and interpret the language.

Here's an example of a basic BNF logic:

<symbol> = _expression_

Where <symbol> is Non-terminal and _expression_ can be Non-terminal or Terminal.

Since there are many different languages we can use to write a program, there needs to be a way for the computer to understand all these different languages. This is done through an interpreter. The interpreter is a program itself, that "translates" the code so that it can be executed by the computer.

(Python is the programming language we'll be using for this course and the notes that follow will be referring to to that language specifically)

Strings and Variables

Variables are used to define and contain values: var_name = expression - where var_name is the identifier and expression is the value. The = means that the right side of the variable goes into the left side, rather than constituting an equal sign. Variables are useful to hold values that might be used repetitively and also give a better understanding of what the code is executing, as long as the variable is given a descriptive identifier.

A string is a sequence of characters. It needs to be contained within either single or double quotes: identifier = "This is a String" is the same as identifier = 'This is a String'. It's important to note that characters are different from a pure number value in the sense that: 2 + 2 gives 4 whereas '2' + '2' gives 22.

Functions

Functions process input and produce an output. You make a function by defining def the number of inputs (parameters) the function will run and then what the function should output return. You can then use the function by running (calling) it, by using print for example, and giving the value(s) for the inputs. It should then output the return you defined.

def functionName(input1,input2,input3,...):

Functions are very useful as they can be called any number of times and also be used within other functions and therefore avoids code repetition.

Evaluate and Loop

You can evaluate expressions with the if statement. The if statement evaluates a test expression as TRUE or FALSE (boolean). If the expression is TRUE, it moves down within its block. If the expression is FLASE it moves on to the next block which can be an else function which defines the execution process in case of FALSE.

To loop blocks we use the while control flow statement. It also has a test expression that it evaluates as TRUE or FALSE. It will continue to run the block(s) until the test statement is FALSE or a break statement becomes TRUE.

Structured Data

Lists can be used to store data, in a similar way as in Variables but in a more structured way. The list is contained within brackets: listName = [string, number, list] Lists can even contain other lists (nested). The data stored in lists are mutable, meaning the stored string or number can be changed. The data stored in lists are indexed in the same way as strings but instead of every character having a number, every list entry has a number starting from 0. date_list = ['day', 'month', 'year'] where day has index 0 date_list[0] and year has index 2 date_list[2].

The append method is used to add data entries to lists (mutate). It will add a new entry to an existing list. date_list.append('century') will add the string 'century' to the previous list. The + operand can be used to concatenate lists and in doing so creating a new list.

The len method can be used to give you the length of a string or list in number of characters or list entries (nested lists will only have 1 number in length). It's important to note however that the output value starts from 1 and not 0, so will not correspond to the index value. len(date_list) would now return 4 but the last entry 'century' has index 3.

For loops are similar to while loops but run for a set number of times as defined by range or until the end of a string or a list. For loops, unlike while loops, don't use a condition statement: for e in p: will loop trough the whole string or list p while for e in range(0,6) will loop from 0 to 5 (6 times).

A methodical way to program

When writing code it's important to structure the task in a methodical way so that you have a clear understanding of what the different elements of the task requires. First of all it's important to define inputs and outputs. It can be a good idea to write some pseudo code to get a rough idea of how you should structure your work. It can also be useful to define a number of test cases to identify different possible scenarios that the code should be able to process and to make sure the the code is executing in the way you envision it to do. You should break down the problem in different areas and then work on one at a time; define the function, test it and the move on to the next one. Sometimes you might have to go back to a function and refine it at a later stage but the general idea is to start with a big picture approach and then zoom in and refine the different parts of the code as you progress.

Program defensively. You should consider who the user of your program will be and write your code to take into account possible errors. Try to output helpful guidance if there's an input error for example.

Another very important aspect of coding is debugging. Again, a methodical approach is essential to locate and fix your code. print, return and test()-functions can be used to make sure that the different functions are giving you the desired output. And remember that the computer reads code literally so simple misspellings and typos will stop the code from executing correctly. It's great to use other programmers solution to save time and to get new ideas but remember to change to code as required to fit into your program.

Stage 3: Object-Oriented Programming (OOP)

Definition

Object-Oriented Programming is a programming concept structured around objects rather than procedures. In OOP the main focus is how data is organized. In class-based OOP, data is stored and structured in classes (as explained below). Objects can modify the internal data (properties) through procedures (methods) encapsulated within the class (more on this below). Another cornerstone in OOP is the interaction between these objects (more on this below).

For a more accurate and elaborate definition please see the Wikipedia atricle on OOP.

Class vs. Object

The first and most important concept to understand in OOP is the distinction and relationship between Class and Object. In Python however, everything is an Object: Classes, Instances, Variables, Lists, Integers, Strings etc. This can be a bit confusing as OOP makes a distinction between Class and Object. It's not a problem though as long as you understand that the reference to an Object, in this context, refers to an Object as an Instance of a Class - something concrete, while Class is something abstract. A Class is a template that contains information specifying what attributes and behavior an Object created from that template will have. Class is the abstract - the idea, and an Instance is an Object of that idea.

Structure

Within classes, variables are referred to as Attributes or Properties and functions as Methods. The data is said to be Encapsulated within the class which is one of the fundamentals of OOP. The classes in turn will be grouped in files referred to as Modules. Modules can be imported into other modules by adding import and the file name without .py as long as the modules are in the same directory. Once a module has been imported, its classes are accessible to construct instances of that class. In Python there are also built-in Modules in the Python Standard Library. These are imported in the same way.

Within a Class the Attributes can be divided into Class Attributes and Instance Attributes (also referred to as Properties). Class Attributes are placed immediately after the class name and the value will be the same for all instances of that class. Next comes the Constructor that initializes the object or instance. This is done through the __init__ method, where the Instance Attributes are placed. These attributes are what defines the instance when created. For example class Person(Human): might have a constructor def __init__(self, name, age, gender) and the Instance Attributes self.name = name, self.age = age and self.gender = gender. Unlike Class Attributes, Instance Attributes are unique to each Instance.

Inheritance

Another fundamental of OOP is inheritance. Just as in HTML and CSS, properties will be inherited depending on the relationships. In OOP classes will inherit Properties and Methods from it's ancestors. class Machine(object) is a parent of class Vehicle(Machine) is a parent of class Car(Vehicle) because Car is-a-type-of Vehicle is-a-type-of Machine is-a-type-of (object). class Car(Vehicle) would have a number of properties defining what an instance of Car would be like. For example there could be a self.engine property. Engine could itself be a class but it wouldn't inherit from class Car(Vehicle) as engine is-not-a-type-of car. We would define the Car - Engine relationship as a has-a relationship.

In the class Machine(object) class Vehicle(Machine) class Car(Vehicle) chain, Vehicle inherits Properties and Methods from Machine, and Car inherits from both ancestors. So if there is a method defined in the class Machine then that method will behave the same in all sub-classes. If the same method were to be defined differently in Vehicle then that would override the method in Machine and behave according to the new definition for Vehicle and its sub-classes. This is called method override.

Concept (and benefits of OOP)

Just as with classes in CSS, classes in Python can be used to store data that you want to be able to access multiple times and therefore eliminate repetition. A Module works much like a CSS-file that you reference in a HTML document where you can then reuse the code you've written in CSS over and over again.

One of the greatest strengths of OOP is how it creates structure and order to programs. As we are able to encapsulate data in classes and group classes in modules, we get a better organization of the code which makes it easier for multiple developers to understand and work together on a project. We can devise, isolate and group data in relevant clusters so that problems solving, debugging and development can be done in parallel.

The idea in OOP is that everything is grouped and organized in a relevant way. Modules, Classes and Objects are separated for structure but at the same time accessible and interconnected for flexibility and function.
